📚 Legal RAG Agent

基于 RAG（Retrieval-Augmented Generation）的法律问答智能体系统

🚀 项目简介

本项目构建了一个面向法律场景的智能问答 Agent，旨在解决大语言模型在专业领域中的幻觉问题（Hallucination），尤其是在法律条文引用不准确这一关键痛点。

系统基于 LangChain + 向量数据库 + 微调大模型（Qwen2.5-3B），通过引入检索增强生成（RAG），实现从“生成式回答”到“基于证据的回答”的转变，使模型输出具备可追溯性与高可信度。

🎯 核心目标

✅ 提升法律问答的准确性与专业性

✅ 解决大模型“编造法律条文”的问题

✅ 实现“基于真实法条生成回答”

✅ 构建可扩展的 Agent + RAG 架构

🛠️ 技术栈
模块	技术
Agent 框架	LangChain
大语言模型	Qwen2.5-3B（LoRA 微调）
向量模型	qwen-embedding
向量数据库	Chroma
数据来源	法律条例结构化数据（subdata）
核心技术	RAG / Prompt Engineering / LoRA
📂 数据处理流程

法律条文按“条”级别进行切分（保证最小语义单元）

使用 embedding 模型进行向量化

存入 Chroma 向量数据库

查询时进行语义相似度检索（Top-K）

⚙️ 核心实现
1️⃣ 微调模型（LoRA）

基于中文法律语料进行微调

提升模型：

法律术语使用能力

法律表达风格

回答结构规范性

👉 但存在问题：

❗ 模型会“编造”法律条文（幻觉严重）

2️⃣ 引入 RAG 机制

通过检索增强生成，解决模型“知识不可靠”问题：

检索相关法律条文（Top-K）

注入 Prompt 作为上下文

强约束模型基于法条回答

3️⃣ Agent 工作流

基于 ReAct 思想：

Query → Thought → 是否检索 → Action（RAG） → Observation → Final Answer
